# 가치평가 프레임워크 — trading-bot 구현 컨텍스트

> Claude 채팅 세션(2026-07-21)에서 정리한 주식 가치평가 프레임워크를
> 코드로 옮기기 위한 설계 문서. Claude Code 세션 시작 시 이 문서를 먼저 읽힐 것.
> 원본은 기업가치 기반 매매 기준에 대한 Q&A이며, 이 문서는 그것을
> 구현 가능한 명세로 재정리한 것이다.

---

## 1. 이 봇이 따르는 판단 규칙

구현 중 판단이 갈릴 때는 아래 5개로 되돌아온다.

1. **가격을 예측하지 않는다.** 가치를 계산하고 현재 가격과 비교한다.
2. **매수단가는 판단 함수의 입력이 아니다.** 현재가만 입력이다. 평단가·수익률·본전 개념은 의사결정 로직 어디에도 들어가지 않는다.
3. **적정가는 단일 값이 아니라 `(보수, 기준, 낙관)` 3-튜플이다.** 스칼라를 반환하는 valuation 함수는 만들지 않는다.
4. **매매 신호는 "현재가에서 남은 연환산 기대수익률(IRR)이 요구수익률을 넘는가"로 낸다.** 상승률·하락률 기준의 트리거는 쓰지 않는다.
5. **수급·차트는 종목 선택이 아니라 분할 집행 타이밍에만 쓴다.** 가치평가 레이어와 실행 레이어를 분리한다.

## 2. 핵심 수식

```
# 잉여현금흐름 (비금융기업)
FCFF = EBIT × (1 − tax_rate) + D&A − CAPEX − ΔWorkingCapital

# 기업가치
EV = Σ[t=1..N] FCFF_t / (1 + WACC)^t  +  TV / (1 + WACC)^N
TV = FCFF_{N+1} / (WACC − g_terminal)

# 주주가치 → 주당가치
EquityValue    = EV − NetDebt − MinorityInterest + NonOperatingAssets
ValuePerShare  = EquityValue / DilutedShares

# 기대 연환산 수익률
IRR = ((P_T + ΣDividends) / P_0)^(1/T) − 1

# 요구수익률을 만족하는 매수가 상한
MaxBuyPrice = (P_T + ΣDividends) / (1 + r_required)^T

# 요구수익률
r_required = r_riskfree + equity_risk_premium + firm_specific_premium

# 성장은 공짜가 아니다 (일관성 검증용 제약)
g ≈ reinvestment_rate × ROIC
```

### 구현 시 반드시 지킬 것

- **통화 일치.** 달러 현금흐름은 달러 할인율, 원화 현금흐름은 원화 할인율. 환율 손익은 가치평가가 아니라 포트폴리오 레이어에서 별도 관리.
- **말기가치 민감도.** TV가 EV의 60~80%를 차지한다. `WACC − g_terminal`이 분모이므로 g를 1%p 올리면 가치가 15~20% 튄다. `g_terminal`은 장기 명목 경제성장률(약 2~3%)을 상한으로 하드 클램프할 것.
- **희석 반영.** 스톡옵션·주식보상·전환사채·유상증자를 `DilutedShares`에 반영. 기본주식수를 쓰면 성장주가 체계적으로 과대평가된다.
- **성장-재투자 일관성 검증.** 고성장을 가정하면서 CAPEX·R&D·운전자본 투입이 낮으면 가정이 모순이다. `g ≈ reinvestment_rate × ROIC` 를 assertion으로 걸 것.

## 3. 모듈 구성 (제안)

```
data/        재무제표 수집·정규화 (DART / EDGAR XBRL), 공시일 기준 타임스탬프 부여
valuation/   DCF 엔진, 잔여이익모형, NAV, SOTP, rNPV — 공통 인터페이스
scenario/    보수·기준·낙관 가정 세트 생성 및 가치 3-튜플 산출
reverse/     역산 DCF — 현재가가 내포하는 성장률·마진 추출
decision/    IRR 계산 및 4구간 행동 신호 출력
news/        LLM 기반 뉴스 3분류 필터
backtest/    공시일 기준 시뮬레이션, 비용·슬리피지 반영
execution/   분할매수 스케줄링 (가치평가와 완전 분리)
```

## 4. 기업 유형별 평가 모델 라우팅

`valuation/` 은 기업 유형에 따라 다른 모델로 분기한다. 모든 종목에 DCF를 돌리면 안 된다.

| 기업 유형 | 주 모델 | 교차검증 | 주의점 |
|---|---|---|---|
| 안정 제조·소비재·서비스 | FCFF DCF | EV/EBIT, PER | 일회성 손익 제거 |
| 은행·보험 | 잔여이익모형 | PBR, 배당할인 | EV/EBITDA 사용 금지 (예금은 부채가 아니라 원재료) |
| 반도체·화학·철강·해운 | 중간경기 이익 기준 | EV/EBITDA | 호황기 저PER은 매수 신호가 아니라 고점 신호 |
| SaaS·플랫폼·고성장 | 다단계 DCF | EV/Sales | 매출배수 단독 판단 금지, 희석 반드시 반영 |
| 리츠·부동산 | NAV | P/FFO, P/AFFO | 감가상각 때문에 순이익이 왜곡됨 |
| 지주회사 | SOTP | 순자산가치 | 지주사 할인, 이중상장 |
| 바이오·임상단계 | rNPV | — | 가치 범위가 매우 넓음, 포지션 축소 |

## 5. 의사결정 함수 명세

```
입력:  current_price, scenario_values(P_T 보수/기준/낙관),
       expected_dividends, T(투자기간), r_required
출력:  signal ∈ {ACCUMULATE, PARTIAL, HOLD_OR_TRIM, EXIT}, 근거 수치
```

각 시나리오의 `MaxBuyPrice`를 계산해 경계로 삼는다.

| 현재가 위치 | 해석 | 신호 |
|---|---|---|
| ≤ MaxBuy(보수) | 최악 가정에서도 요구수익률 충족 | ACCUMULATE |
| MaxBuy(보수) ~ MaxBuy(기준) | 기준은 충족, 보수는 미달 | PARTIAL |
| MaxBuy(기준) ~ MaxBuy(낙관) | 낙관 시나리오에 의존 | HOLD_OR_TRIM |
| > MaxBuy(낙관) | 낙관 가정에서도 미달 | EXIT |

### 추가매수 게이트 (3조건 AND)

가격 조건만으로 추가매수하지 않는다. 세 조건이 모두 참일 때만 통과.

1. **가격**: 현재가에서 기준 또는 보수 IRR ≥ r_required
2. **투자 논리 유지**: 시장점유율 / 정상 마진 / ROIC / 재무구조 / 현금흐름 창출력이 훼손되지 않음
3. **생존 가능성**: 차입금 만기, 이자보상배율, 현금소진 속도, 증자 가능성, 재무약정 — 가치 회복 전에 희석되거나 부도날 위험이 낮음

## 6. 역산 DCF (스크리닝 엔진의 핵심)

순DCF는 가정을 넣어야 답이 나오므로 확증편향에 취약하다. 역산은 반대 방향으로 푼다.

```
f(g) = DCF_value(g, 나머지 가정 고정) − current_price = 0
```

이분법 또는 뉴턴법으로 근을 구하면 "시장이 현재 가격에 내포시킨 성장률"이라는 스칼라가 나온다. 전 종목에 대해 계산하면 정렬 가능한 스크리닝 지표가 되고, 이를 과거 실적·업종 구조와 비교해 현실성을 판정한다.

같은 방식으로 내포 영업이익률, 내포 ROIC도 뽑을 수 있다. 봇 자동화에는 순DCF보다 이쪽이 훨씬 실용적이다.

## 7. LLM 사용 규칙

- **숫자 계산은 절대 LLM에 맡기지 않는다.** DCF·IRR·근 찾기는 전부 결정론적 코드로.
- **LLM은 비정형 텍스트 → 구조화된 변수 추출에만 쓴다.** 사업보고서에서 세그먼트별 매출 추출, 경영진 가이던스 파싱 등.
- **뉴스 3분류 필터** — 뉴스가 가치평가 변수를 실제로 바꾸는지만 판정한다.

```
분류: 이 뉴스는 다음 중 무엇을 바꾸는가?
  A. 미래 현금흐름의 규모 또는 시점
  B. 무위험금리 / 할인율
  C. 위험프리미엄 (부채, 희석, 부도확률 포함)
  D. 위 어느 것도 아님

D이면 적정가를 갱신하지 않고 단기 수급 이벤트로 로깅만 한다.
A/B/C이면 해당 가정 변수를 재추정 큐에 넣는다.
```

## 8. 백테스트 필수 요건

이 세 가지를 처리하지 않은 백테스트 결과는 신뢰하지 않는다.

1. **Look-ahead bias** — 재무 데이터는 회계기간이 아니라 **공시일** 기준으로 정렬해야 한다. 2025년 4분기 실적은 2026년 2~3월 이후에야 알 수 있다. 이 하나만 틀려도 백테스트 수익률이 비현실적으로 좋아진다.
2. **Survivorship bias** — 상장폐지·합병으로 사라진 종목을 유니버스에서 빼면 성과가 과대평가된다. 델리스팅 이력을 포함할 것.
3. **거래 마찰** — 수수료, 세금(거래세·배당세), 슬리피지, 호가 단위, 유동성 제약을 반영.

추가로: 재무 데이터는 소급 수정(restatement)이 발생하므로, 가능하면 point-in-time 스냅샷을 보관한다.

## 9. 데이터 소스

- **국내**: DART OpenAPI (재무제표·주요사항·지분공시), KRX (시세·투자자별 매매동향)
- **미국**: SEC EDGAR — 10-K / 10-Q / 8-K, XBRL 재무 태그 (표준화되어 파싱 용이)
- **업종 벤치마크**: 자본비용·PBR·EV/EBITDA·ROIC 업종 평균 (Damodaran 연간 데이터셋 등). 단, 업종 평균은 적정가가 아니라 **내 가정의 현실성을 점검하는 기준선**으로만 쓴다.

## 10. 아직 정해지지 않은 것

Claude Code 세션에서 먼저 결정해야 할 항목들.

- [ ] `r_required` 값과 산정 방식 (고정값? 종목별 위험 반영?)
- [ ] 투자기간 `T` (문서 예시는 3년)
- [ ] 대상 유니버스 (국내만? 미국 포함? 시가총액 하한?)
- [ ] 재평가 주기 (분기 실적 발표 기준? 고정 스케줄?)
- [ ] 포지션 사이징 규칙 (가치 범위 폭에 반비례?)
- [ ] 실행 방식 (신호만 알림? 자동 주문? 증권사 API 약관 확인 필요)
- [ ] 페이퍼 트레이딩 검증 기간

## 11. 종목별 상태 스키마

각 종목에 대해 분기마다 갱신할 레코드 구조.

```
ticker
company_type          # 라우팅 키 (4장 표)
primary_model         # DCF / RIM / NAV / SOTP / rNPV
normalized_financials # 정상 매출, 마진, ROIC, FCF
financial_risk        # 순차입금, 만기, 이자비용, 희석 가능성
assumptions           # 보수/기준/낙관 × (성장률, 마진, 재투자, 할인율)
value_range           # (V_low, V_base, V_high)
implied_assumptions   # 역산 DCF 결과 — 현재가가 내포한 성장률·마진
expected_irr          # 시나리오별 T년 IRR
buy_zone / trim_zone / exit_condition
next_review_date      # 분기실적, 주요 공시, 산업 데이터 발표일
```

## 면책

이 문서는 가치평가 개념을 코드로 옮기기 위한 기술 명세이며 투자 자문이 아니다.
백테스트 성과가 실전 성과를 보장하지 않는다. 실제 자금 투입 전 충분한 기간의
페이퍼 트레이딩과 증권사 API 이용약관·시세 데이터 라이선스 검토가 필요하다.
