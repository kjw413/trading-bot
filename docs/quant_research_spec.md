# 퀀트 연구 기준 명세 (Quant Research Spec)

이 문서는 `trading-bot` 퀀트 연구의 대상, 벤치마크, 검증 방식, 전략 승격 기준을 정의한다.
설정값의 단일 원본은 [`config/research.toml`](../config/research.toml)이며, 이 문서는 그 근거와 규칙을 설명한다.

관련 실행 계획: [`trading_bot_agentic_ai_execution_plan_260714.md`](../trading_bot_agentic_ai_execution_plan_260714.md) (M6)

## 1. 연구 대상

### 1.1 1단계 전략: ETF 모멘텀 로테이션

첫 번째 실전 후보 전략은 **미국 상장 ETF 모멘텀 로테이션**으로 정의한다.

- 미국 ETF를 1단계 대상으로 삼는 이유:
  - 상장 기간이 길어 In-sample / Out-of-sample 구분이 가능하다 (2007년 이후 데이터 확보 가능).
  - 유동성이 높아 슬리피지 가정이 단순하다.
  - 자산군(주식/채권/원자재/부동산) 분산이 한 시장 안에서 가능하다.
- 한국 상장 ETF 유니버스는 2단계로 확장한다 (다수 종목의 상장일이 2020년 이후라 검증 기간이 짧다).

### 1.2 투자 가능 유니버스 (US ETF)

| 자산군 | 심볼 | 설명 |
|---|---|---|
| 미국 대형주 | SPY | S&P 500 |
| 미국 기술주 | QQQ | Nasdaq 100 |
| 미국 소형주 | IWM | Russell 2000 |
| 선진국 주식 | EFA | MSCI EAFE |
| 신흥국 주식 | EEM | MSCI Emerging Markets |
| 미국 장기국채 | TLT | 20+ Year Treasury |
| 미국 중기국채 | IEF | 7-10 Year Treasury |
| 투자등급 회사채 | LQD | Investment Grade Corporate |
| 금 | GLD | Gold |
| 원자재 | DBC | Commodity Index |
| 미국 리츠 | VNQ | US REIT |

- 유니버스 변경은 이 문서와 `config/research.toml`을 함께 수정하고 커밋에 사유를 남긴다.
- 유니버스에 새 ETF를 추가할 때는 최소 상장기간(아래 필터 기준)을 충족해야 한다.

### 1.3 안전자산 (절대 모멘텀 미충족 시 대피처)

- 기본: `IEF` (미국 중기국채)
- 절대 모멘텀 필터를 통과한 ETF가 선정 수보다 적으면, 부족분은 안전자산으로 배분한다.

## 3. 전략 정의 (초기값)

```text
신호 계산일: 매월 마지막 거래일 (월말 종가 기준)
집행일:      다음 달 첫 거래일 시가 (룩어헤드 방지 흐름 유지)

모멘텀 점수:
score = 0.2 × 3개월 수익률 + 0.3 × 6개월 수익률 + 0.5 × 12개월 수익률

필터 (모두 통과해야 후보):
- 가격 > 200일 이동평균 (절대 모멘텀)
- 최소 상장기간 ≥ 400 거래일 (12개월 모멘텀 + 여유분)
- 20일 평균 거래대금 ≥ 유동성 기준

선정: 필터 통과 ETF 중 점수 상위 3개
비중: 변동성 역가중 (60일 변동성 역수 비례), 대안으로 동일비중
주기: 월 1회
```

- 파라미터의 정확한 값은 `config/research.toml`의 `[strategy.etf_momentum]`을 따른다.
- 위 초기값은 In-sample 구간에서만 조정할 수 있다 (5장 참조).

## 4. 실험 기간 구분

| 구간 | 기간 | 용도 |
|---|---|---|
| In-sample | 2010-01-01 ~ 2018-12-31 | 전략 설계, 파라미터 탐색 |
| Validation | 2019-01-01 ~ 2021-12-31 | 파라미터 확정, 모델 선택 |
| Out-of-sample | 2022-01-01 ~ 현재 | **최종 평가 전용. 절대 파라미터 조정에 사용 금지** |

규칙:

1. Out-of-sample 구간의 성과를 보고 파라미터·유니버스·필터를 바꾸는 것을 금지한다.
2. Out-of-sample 평가는 전략 버전당 원칙적으로 1회만 수행하고, 결과를 실험 기록에 남긴다.
3. Walk-forward 검증은 In-sample + Validation 구간 안에서 수행한다.
4. 유니버스 내 일부 ETF(DBC 2006년, VNQ 2004년 상장 등)의 데이터 시작일이 달라도
   구간 정의는 고정한다. 상장 전 종목은 해당 날짜 유니버스에서 제외될 뿐이다.

## 5. 검증 방식

### 5.1 공통 규칙

- 모든 백테스트는 거래비용(`config/default.toml`의 `[fees]`)과 슬리피지를 반영한다.
- 종가 신호는 반드시 다음 거래일 시가 체결로 처리한다.
- 상장폐지·상장 전 기간을 유니버스에서 제외하는 방식으로 생존자 편향을 관리한다.
- 실험은 Git commit, 데이터 버전, 파라미터와 함께 기록한다.

### 5.2 필수 검증 항목

1. 벤치마크 대비 초과수익 (연율화)
2. In-sample / Validation / Out-of-sample 구간별 분리 성과
3. Walk-forward: 학습 3년 → 검증 1년, 1년 단위 전진
4. 파라미터 민감도: 핵심 파라미터 ±1 단계(예: 상위 2/3/4개, 가중치 조합 변경)에서 성과 부호 유지
5. 거래비용 민감도: 비용 2배 가정에서도 벤치마크 초과 논리 유지
6. 시장 국면별 성과: 벤치마크 상승장/하락장(200일 이평 기준) 구분 성과
7. 집중도: 단일 ETF 제외 시(leave-one-out) 전략이 무너지지 않을 것

## 6. 전략 승격 기준

전략은 아래 단계를 순서대로 통과해야 다음 단계로 승격된다.

```text
연구(In-sample) → 검증(Validation/Walk-forward) → Out-of-sample 평가
→ 모의투자 → 소액 실전 → 단계적 확대
```

### 6.1 백테스트 → 모의투자 승격 기준

Out-of-sample 및 Walk-forward 집계 기준으로 아래를 모두 충족:

| 항목 | 기준 |
|---|---|
| 벤치마크 대비 초과수익 | > 0 (연율화, 비용 반영 후) |
| Sharpe Ratio | ≥ 0.5 |
| 최대 낙폭 (MDD) | ≤ 25% (목표 20%) |
| 연간 회전율 | ≤ 600% (편도 기준) |
| 파라미터 민감도 | 주변 파라미터에서 성과 부호 유지 |
| 비용 2배 민감도 | 초과수익 > 0 유지 |
| Walk-forward 구간 승률 | 검증 구간의 60% 이상에서 벤치마크 초과 또는 MDD 우위 |

### 6.2 모의투자 → 소액 실전 승격 기준 (M15/M18과 연동)

- 모의투자 20거래일 이상 상태 손상 없음
- 중복 주문 0건, 포지션 불일치 0건, 휴장일 주문 0건
- 재시작 후 상태 복구 성공
- 백테스트 신호와 모의투자 신호 일치
- 목표 비중 대비 실제 비중 오차가 허용범위(±2%p) 이내

### 6.3 강등 기준

실전·모의 운영 중 아래 발생 시 전략을 중단하고 연구 단계로 되돌린다.

- MDD가 백테스트 Out-of-sample MDD의 1.5배 초과
- 12개월 롤링 수익률이 벤치마크를 10%p 이상 하회
- 데이터·주문 오류로 인한 신호 불일치 반복 (월 2회 이상)

## 8. 용어

- **회전율(Turnover)**: 기간 중 `min(매수금액, 매도금액) / 평균 순자산`의 합. 연율화하여 보고한다.
- **IC (Information Coefficient)**: 팩터 점수와 미래 수익률의 Spearman 순위상관.
- **Walk-forward**: 학습 구간에서 파라미터를 정하고 바로 다음 구간에서 평가하며 앞으로 이동하는 검증.
